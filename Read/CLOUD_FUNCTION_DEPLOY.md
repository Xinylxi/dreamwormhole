# 云函数部署指南

## 📍 查找云函数部署选项

如果在右键菜单中找不到"上传并部署"选项，请尝试以下方法：

## 方法一：通过工具栏菜单部署

1. **选中云函数文件夹**
   - 在左侧文件树中，点击 `cloudfunctions/coze-proxy` 文件夹

2. **使用顶部菜单**
   - 点击顶部菜单栏 `工具` 或 `云开发`
   - 查找以下选项之一：
     - `上传并部署：云端安装依赖`
     - `上传并部署`
     - `部署云函数`
     - `上传云函数`

## 方法二：通过云开发控制台部署

1. **打开云开发控制台**
   - 点击开发者工具顶部工具栏的 **`云开发`** 按钮
   - 或在菜单栏选择：`工具` → `云开发`

2. **进入云函数管理**
   - 在云开发控制台中，点击左侧菜单的 **`云函数`**

3. **找到 coze-proxy 函数**
   - 在函数列表中找到 `coze-proxy`
   - 点击函数名称进入详情页

4. **上传代码**
   - 点击 **`上传代码`** 或 **`重新部署`** 按钮
   - 选择本地 `cloudfunctions/coze-proxy` 目录
   - 点击上传

5. **安装依赖**
   - 上传后，点击 **`安装依赖`** 或等待自动安装
   - 查看部署日志确认成功

## 方法三：命令行部署（如果已安装 CLI）

```bash
# 进入项目目录
cd "E:\小程序前端\WeChatProjects\miniprogram-1"

# 进入云函数目录
cd cloudfunctions/coze-proxy

# 安装依赖
npm install

# 回到项目根目录，使用微信开发者工具 CLI 部署
# （需要先安装微信开发者工具 CLI）
```

## 方法四：手动上传文件

如果以上方法都不行，可以手动操作：

1. **准备文件**
   - 确保 `cloudfunctions/coze-proxy/` 目录包含：
     - `index.js`
     - `package.json`
     - `node_modules/`（可选，会在云端安装）

2. **打包文件**
   - 将 `index.js` 和 `package.json` 打包成 zip 文件
   - 注意：不要包含 `node_modules`（会在线安装）

3. **上传到云开发控制台**
   - 打开云开发控制台 → 云函数
   - 选择 `coze-proxy`
   - 点击 "上传代码"
   - 选择打包的 zip 文件

## 方法五：通过代码编辑器部署

某些版本的微信开发者工具支持：

1. 在代码编辑器中右键点击 `index.js`
2. 查找 `上传`、`部署` 或 `云函数` 相关选项

## 🔍 检查当前工具版本

不同版本的微信开发者工具界面可能不同：

- **新版工具**（3.x）：通常在右键菜单或工具栏
- **旧版工具**（2.x）：可能需要通过云开发控制台

## ✅ 验证部署是否成功

部署完成后，检查：

1. **查看云开发控制台日志**
   - 云开发控制台 → 云函数 → `coze-proxy` → `日志`
   - 应该看到最近的部署日志

2. **测试云函数**
   - 在小程序中调用云函数
   - 查看控制台是否有新的日志

3. **检查函数状态**
   - 云开发控制台 → 云函数 → `coze-proxy`
   - 状态应该显示为"已部署"或"运行中"

## 🚨 常见问题

### Q1: 找不到"云开发"按钮？

**解决方案：**
- 确认已开通云开发
- 在 `app.js` 中已初始化云开发
- 可能需要重新打开开发者工具

### Q2: 云函数列表中没有 coze-proxy？

**解决方案：**
1. 首先在本地创建云函数：
   - 右键 `cloudfunctions` 文件夹
   - 选择 `新建 Node.js 云函数`
   - 命名为 `coze-proxy`
   
2. 或手动创建：
   - 在 `cloudfunctions` 下创建 `coze-proxy` 文件夹
   - 创建 `index.js` 和 `package.json`

### Q3: 部署失败？

**检查清单：**
- [ ] `package.json` 格式正确
- [ ] `index.js` 文件存在且语法正确
- [ ] 云开发环境已正确配置
- [ ] 网络连接正常

### Q4: 如何查看部署进度？

1. 打开云开发控制台
2. 点击 `云函数` → `coze-proxy`
3. 查看 `版本管理` 或 `部署日志`

## 📝 快速检查清单

部署前确认：
- [ ] 云开发已开通
- [ ] `app.js` 中已配置云环境 ID
- [ ] `cloudfunctions/coze-proxy/index.js` 文件存在
- [ ] `cloudfunctions/coze-proxy/package.json` 文件存在
- [ ] API Key 已更新为最新值

部署后验证：
- [ ] 云开发控制台显示部署成功
- [ ] 可以正常调用云函数
- [ ] 控制台日志无错误
