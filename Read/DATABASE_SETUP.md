# 数据库配置和索引设置

## 数据库集合

本项目使用以下两个云数据库集合：

### 1. dreams（梦境记录）
存储用户的梦境记录

**字段结构：**
- `_id`: 记录ID（自动生成）
- `_openid`: 用户openid（自动生成）
- `content`: 梦境内容
- `date`: 日期
- `time`: 时间
- `emotions`: 情绪标签数组
- `tags`: 标签数组
- `interpretation`: 解析结果
- `planetType`: 星球类型
- `planetName`: 星球名称
- `emoji`: 表情符号
- `color`: 颜色
- `position`: 位置坐标
- `createdAt`: 创建时间（服务器时间）
- `updatedAt`: 更新时间（服务器时间）

### 2. planets（星球）
存储用户的梦境星球

**字段结构：**
- `_id`: 记录ID（自动生成）
- `_openid`: 用户openid（自动生成）
- `name`: 星球名称
- `emoji`: 表情符号
- `color`: 颜色
- `description`: 描述
- `position`: 位置坐标
- `dreamIds`: 关联的梦境ID数组
- `dreamCount`: 梦境数量
- `keywords`: 关键词数组
- `representativeDream`: 代表性梦境内容
- `lastDreamDate`: 最后一次梦境日期
- `createdAt`: 创建时间（服务器时间）
- `updatedAt`: 更新时间（服务器时间）

## 必需的数据库索引

为了优化查询性能，需要为 `dreams` 集合创建以下索引：

### 索引 1: 用户梦境列表查询优化
**用途：** 优化获取用户梦境列表的查询性能

**索引字段：**
- `_openid`: 升序 (asc)
- `createdAt`: 降序 (desc)

**创建方法：**

#### 方法一：通过云开发控制台创建（推荐）
1. 登录微信开发者工具
2. 打开「云开发」控制台
3. 进入「数据库」→「dreams」集合
4. 点击「索引管理」→「添加索引」
5. 添加以下字段：
   - 字段名: `_openid`，排序: `升序`
   - 字段名: `createdAt`，排序: `降序`
6. 保存索引

#### 方法二：使用控制台提供的快速创建链接
控制台日志中会提供快速创建索引的链接，点击即可自动创建。

**查询示例：**
```javascript
db.collection('dreams')
  .where({
    _openid: '{用户openid}' // 自动填充
  })
  .orderBy('createdAt', 'desc')
  .limit(10)
  .get()
```

## 性能优化建议

1. **使用索引查询**：确保所有频繁查询都使用了索引字段
2. **限制返回数量**：使用 `.limit()` 限制返回的记录数
3. **避免全表扫描**：查询时尽量使用 `_openid` 等索引字段作为查询条件
4. **定期清理数据**：删除过期或不需要的记录
5. **使用服务器时间**：使用 `db.serverDate()` 确保时间的一致性

## 数据库权限配置

建议在云开发控制台设置集合权限：

- **dreams 集合**：仅创建者可读写
- **planets 集合**：仅创建者可读写

这样可以确保用户只能访问自己的数据。

2. 关注慢查询日志
3. 根据实际使用情况调整索引配置
